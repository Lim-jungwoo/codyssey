# 🐍 Python 대표 웹 프레임워크 3가지 비교

Python에는 다양한 웹 프레임워크가 있지만, 가장 널리 쓰이는 세 가지는 다음과 같다:

- **Flask**
- **Django**
- **FastAPI**

---

## 1. Flask

| 항목 | 설명 |
|------|------|
| 특징 | 가볍고 유연한 마이크로 프레임워크 |
| 학습 난이도 | 쉬움 |
| 사용 방식 | 필요한 라이브러리만 직접 선택하여 조립 |
| 사용 예시 | 스타트업 MVP, 소규모 API 서버 |
| 주요 특징 | 자유도 높음, 구조 제한 없음 |
| 주요 단점 | 구조 설계나 확장성은 직접 구현 필요 |

---

## 2. Django

| 항목 | 설명 |
|------|------|
| 특징 | 배터리 포함(batteries included) 프레임워크 |
| 학습 난이도 | 중간~높음 |
| 사용 방식 | 기본 제공 기능이 많아 바로 앱 개발 가능 |
| 사용 예시 | 대규모 웹사이트, 관리자 페이지 포함한 웹서비스 |
| 주요 특징 | ORM, 관리자 페이지, 인증, 폼 등 기본 포함 |
| 주요 단점 | 유연성 낮음, 커스터마이징 복잡할 수 있음 |

---

## 3. FastAPI

| 항목 | 설명 |
|------|------|
| 특징 | 최신 Python 타입 힌트를 활용한 비동기 API 프레임워크 |
| 학습 난이도 | Flask보다 조금 높음 |
| 사용 방식 | 타입 기반 요청/응답 유효성 검증 자동화 |
| 사용 예시 | 고성능 REST API, 머신러닝 서비스 API |
| 주요 특징 | 비동기 지원, 자동 문서(Swagger), 빠른 속도 |
| 주요 단점 | 상대적으로 생태계 작고, 학습 곡선 존재 |

---

## ✅ 요약 비교

| 항목     | Flask            | Django              | FastAPI             |
|----------|------------------|---------------------|---------------------|
| 규모     | 소형~중형         | 중형~대형            | 중형~대형            |
| 철학     | 최소한의 것만 제공 | 모든 기능 포함        | 타입 기반 API 자동화 |
| 구조 자유도 | 매우 높음         | 낮음 (고정 구조)     | 중간                |
| 비동기 지원 | X (가능하지만 복잡) | 제한적               | ✅ 완전 지원         |
| 사용 예  | 빠른 프로토타입    | 전통적인 웹 앱       | 고성능 API 서버      |

---

## 🔚 어떤 걸 써야 할까?

- **빠른 실험/작은 API 서버** → Flask
- **관리자 페이지/DB 중심의 웹서비스** → Django
- **비동기 + 자동 문서 + API 중심 서비스** → FastAPI
